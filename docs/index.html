Gemini quality gate for Claude Code

Make Claude prove it.

Bouncer intercepts Claude Code right before it stops, sends the actual work context to Gemini, and blocks the response until the output is verified.

Open Repository Install in 60 seconds
Automatic: runs as a Stop hook on every Claude response.
On-demand: use /bouncer or /bouncer deep for spot checks and final reviews.
Safe by default: fails open on API errors, logs decisions, and handles concurrent invocations.

Hook

Claude tries to stop. Bouncer reads the transcript, tool output, git diff, CLAUDE.md, and active workplan, then asks Gemini to score the result from 1 to 10.

Quick + Deep

Use the automatic Stop hook every time, or call /bouncer for a fast audit and /bouncer deep when you want an independent tool-using reviewer.

Trust Signals

MIT licensed, tested in CI, clean-home installer smoke tested, and explicit about which data gets sent to the Gemini API.

What the block looks like

$ claude "finish the fix and verify it"
========================================
BOUNCER AUDIT: 8/10
========================================

SCORE: 8/10
ISSUES:
- claimed tests ran, but no command output was present
- fix path on empty transcript was not verified
VERDICT: FAIL

Claude keeps working until Gemini approves.

Install

Copy one command, set your API key, and Bouncer will register the Stop hook plus the on-demand skill.

curl -fsSL https://raw.githubusercontent.com/buildingopen/bouncer/master/install.sh | bash
export GEMINI_API_KEY="your-gemini-api-key"
Automatic

Runs on every Stop hook invocation and blocks low-confidence output.

/bouncer

Fast audit using the diff, Claude summary, and project rules.

/bouncer deep

Gemini reads files, runs commands, checks git history, and verifies claims.

Fails open

No API key or API outage means Claude is not permanently blocked.

What gets sent to Gemini

Bouncer is explicit about the audit payload. It can send Claude's assistant response, recent user messages from the transcript, tool call output, git diff, project CLAUDE.md, and the active workplan. Review Google's data handling policies before using it on sensitive projects.
